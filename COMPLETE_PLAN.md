# المرحلة الأولى: تجهيز الأدوات
# ═══════════════════════════════

## 🔑 الأداة 1: إنشاء بوت Telegram والحصول على TOKEN

**الوقت المطلوب:** 5 دقائق

### الخطوات:

**1.** افتح Telegram على موبايلك أو الكمبيوتر

**2.** في شريط البحث اكتب: `@BotFather` واضغط عليه

**3.** اضغط **START** أو ابعت `/start`

**4.** ابعت الأمر: `/newbot`

**5.** BotFather هيسألك: **"What's the name of your bot?"**
   - اكتب أي اسم مثلاً: `EGX Stock Analyzer`

**6.** هيسألك: **"What username?"**
   - لازم ينتهي بـ `bot` مثلاً: `my_egx_stocks_bot`
   - لو الاسم مأخود جرب: `egx_analyzer_bot` أو `cairo_stocks_bot`

**7.** هيديك رسالة فيها **TOKEN** يشبه كده:
   ```
   7123456789:AAFabcdefghijklmnopqrstuvwxyz123456
   ```

**8.** ✅ احتفظ بالـ TOKEN ده - هتحتاجه بعدين

> ⚠️ **تحذير:** متشارش الـ TOKEN مع حد - ده مفتاح البوت بتاعك

---

## 🔑 الأداة 2: OpenAI API Key

**عندك بالفعل ✅** - جهّزه عشان تحطه بعدين

لو محتاج تعمل واحد جديد:
1. روح: https://platform.openai.com/api-keys
2. اضغط **"Create new secret key"**
3. انسخه واحتفظ بيه (بيظهر مرة واحدة بس!)

---

## 🔑 الأداة 3: حساب GitHub

**الوقت المطلوب:** 3 دقائق

**1.** روح: https://github.com

**2.** اضغط **"Sign up"**

**3.** أدخل:
   - Email
   - Password
   - Username (مثلاً: `ahmed_dev` أو أي اسم)

**4.** أكّد الـ email

**5.** ✅ حسابك جاهز

# المرحلة الثانية: رفع الكود
# ═══════════════════════════════

## 📁 إنشاء Repository على GitHub

**الوقت المطلوب:** 2 دقيقة

**1.** بعد تسجيل الدخول على GitHub، اضغط على **"+"** في الأعلى يمين

**2.** اختار **"New repository"**

**3.** في **"Repository name"** اكتب: `egx-bot`

**4.** اختار **"Public"**

**5.** اضغط **"Create repository"** ✅

---

## 📤 رفع الملفات

**الوقت المطلوب:** 5 دقائق

لديك 5 ملفات محتاج ترفعهم:
```
✅ bot.py
✅ requirements.txt
✅ railway.toml
✅ Procfile
✅ runtime.txt
```

### طريقة الرفع:

**1.** افتح الـ repository اللي عملته

**2.** هتلاقي رابط **"uploading an existing file"** - اضغط عليه

**3.** اسحب الـ 5 ملفات أو اضغط **"choose your files"** واختارهم

**4.** تحت في **"Commit changes"** اضغط الزر الأخضر

**5.** ✅ الملفات اترفعت على GitHub

# ═══════════════════════════════
# المرحلة الثالثة: الرفع على السحابة
# ═══════════════════════════════

## ☁️ إعداد Railway - البوت يشتغل 24/7

**الوقت المطلوب:** 10 دقائق

### الخطوة 1: إنشاء حساب Railway

**1.** روح: https://railway.app

**2.** اضغط **"Login"**

**3.** اختار **"Login with GitHub"** ← الأسهل لأن حسابك موجود

**4.** وافق على الصلاحيات

**5.** ✅ دخلت على Railway

---

### الخطوة 2: ربط الكود بـ Railway

**1.** في الصفحة الرئيسية اضغط **"New Project"**

**2.** اختار **"Deploy from GitHub repo"**

**3.** لو طلب منك **"Configure GitHub App"** اضغط عليها ووافق

**4.** هتشوف قائمة بالـ repositories بتاعتك - اختار **`egx-bot`**

**5.** اضغط **"Deploy Now"**

**6.** Railway هيبدأ يبني البوت - هتشوف شاشة logs فيها حاجات بتتكتب

> ⚠️ **طبيعي** يظهر خطأ دلوقتي لأننا لسه ماضفناش الـ API Keys

---

### الخطوة 3: إضافة الـ API Keys ← المهم جداً!

**1.** في Railway، اضغط على الـ project بتاعك

**2.** اضغط على **"Variables"** من القائمة الجانبية

**3.** اضغط **"New Variable"** وأضف الأول:
   ```
   Name:  TELEGRAM_TOKEN
   Value: ضع TOKEN من BotFather هنا
   ```
   اضغط **"Add"**

**4.** اضغط **"New Variable"** تاني وأضف:
   ```
   Name:  OPENAI_API_KEY
   Value: sk-proj-xxxxxxxxxx
   ```
   اضغط **"Add"**

**5.** Railway هيعمل **Redeploy** أوتوماتيك بعد إضافة الـ Variables ✅

---

### الخطوة 4: التأكد إن البوت شغال

**1.** في Railway اضغط على **"Deployments"**

**2.** اضغط على أحدث deployment

**3.** اضغط **"View Logs"**

**4.** لو شفت:
   ```
   🚀 بيشتغل البوت...
   ✅ البوت شغال! انتظر الرسائل...
   ```
   **يبقى شغال! 🎉**

**5.** افتح Telegram، روح للبوت بتاعك، ابعت `/start`
